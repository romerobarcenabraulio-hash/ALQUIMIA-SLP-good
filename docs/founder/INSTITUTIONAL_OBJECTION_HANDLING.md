# Institutional Objection Handling

**Uso:** respuestas sobrias para objeciones institucionales. No usar como copy publicitario.

## "Esto esta validado oficialmente?"

No necesariamente. ALQUIMIA distingue entre dato oficial, dato validado, inferencia, benchmark, estimacion y brecha critica. Si el claim no trae fuente oficial, estudio local o validacion humana, debe tratarse como preliminar.

Requiere validacion humana cuando se quiera usar en Cabildo, contrato, presupuesto, gate o documento oficial.

## "ALQUIMIA certifica resultados?"

No. ALQUIMIA estructura evidencia, calcula escenarios y prepara trazabilidad. No certifica resultados salvo que exista un certificador externo o autoridad competente que emita esa validacion.

## "Quien aprobo esta recomendacion?"

Debe existir una persona responsable. flujo de borradores asistidos, capa de aprendizaje supervisado y otros sistemas internos pueden preparar recomendaciones, pero no aprueban, firman ni cierran gates. Si no hay responsable humano, la recomendacion queda pendiente de revision.

## "Por que capa de aprendizaje supervisado sugiere esto?"

Solo debe sugerir si el patron cumple opt-in, N suficiente, bias check, founder gate, standards check y trazabilidad. Si cualquiera falta, la sugerencia no debe publicarse al cliente.

capa de aprendizaje supervisado no decide; el humano acepta, ajusta o rechaza.

## "Esto es dato municipal o metropolitano?"

Debe identificarse la unidad territorial. Si viene de ZM, se muestra como contexto regional o benchmark, no como cifra municipal. Para afirmar algo municipal se requiere fuente municipal o estudio local.

## "Donde esta el estudio local?"

Si existe, debe mostrarse alcance, fecha, metodo, fuente, responsable y limitaciones. Si no existe, se declara brecha critica o estudio requerido. No se reemplaza con benchmark, inferencia o narrativa.

## "Que pasa si el concesionario no esta de acuerdo?"

Se solicita evidencia operativa: bitacoras, GPS, pesajes, reportes, contrato, capacidad o mantenimiento. Si la evidencia es suficiente, se corrige el dato. Si no hay evidencia local suficiente, el claim queda preliminar o bloqueado.

## "Que tan precisas son las estimaciones?"

La precision depende de la calidad de fuentes, supuestos y estudios locales. Una estimacion no debe presentarse como oficial ni como garantia. Debe mostrar fuente, fecha, metodo, confianza y sensibilidad cuando aplique.

## "Puedo presentar esto en Cabildo?"

Puede presentarse como lectura preliminar o borrador si las advertencias son visibles. Para presentarlo como afirmacion institucional, acuerdo, dictamen o base presupuestal se requiere evidencia minima, responsable humano y revision juridica/técnica segun el claim.

## "Esto sustituye a un consultor o auditor?"

No. ALQUIMIA organiza evidencia, acelera diagnostico, prepara borradores y muestra brechas. No sustituye validacion humana, auditoria externa, estudio local, firma juridica ni decision de autoridad.

## "Que pasa si falta informacion?"

La plataforma debe mostrar pendiente, brecha critica o baja confianza. No debe llenar el vacio con una afirmacion positiva. El siguiente paso es cargar fuente, ejecutar estudio local o degradar el claim.
