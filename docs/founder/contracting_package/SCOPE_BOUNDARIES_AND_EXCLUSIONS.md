# Scope Boundaries and Exclusions

**Uso:** limites y exclusiones que deben acompanar cualquier propuesta preliminar.

## Exclusiones no negociables

- ALQUIMIA no certifica resultados oficiales.
- ALQUIMIA no sustituye auditoria externa.
- ALQUIMIA no sustituye decision publica.
- ALQUIMIA no valida automaticamente datos de terceros.
- capa de aprendizaje supervisado no aprueba, firma ni decide.
- flujo de borradores asistidos no firma, aprueba ni cierra gates.
- Benchmarks no son estudios locales.
- Estimaciones no son datos oficiales.
- Inferencias no son datos validados.
- Municipio y ZM no se mezclan.
- Faltantes de informacion son brechas criticas si afectan claims institucionales.
- Gates y decisiones politicas son humanas.
- Sin opt-in, datos tenant no alimentan analytics agregada.

## Uso obligatorio

Estas exclusiones deben aparecer o estar anexas antes de propuesta, SOW o contrato. Si el cliente pide eliminar una exclusion para sostener un claim no probado, el avance queda bloqueado o requiere decision founder/juridico.
