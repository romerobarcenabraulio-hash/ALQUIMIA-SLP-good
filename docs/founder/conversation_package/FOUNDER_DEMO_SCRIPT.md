# Founder Demo Script

**Uso:** guion operativo para demo institucional. No es pitch deck ni copy publico.

## Apertura de 60 segundos

> ALQUIMIA ayuda a un municipio a ordenar evidencia, escenarios y decisiones de RSU sin convertir datos preliminares en verdad oficial. La plataforma separa validacion, planeacion y ejecucion; muestra fuente, fecha, metodo y confianza; y bloquea claims cuando falta evidencia. Lo que hoy vamos a ver no sustituye Cabildo, juridico, auditoria ni estudios de campo: prepara una conversacion institucional mejor sustentada.

## Que es ALQUIMIA

ALQUIMIA es una plataforma institucional para estructurar diagnostico, planeacion, evidencia, gates humanos y documentacion trazable por tenant municipal.

## Que no es ALQUIMIA

No es certificador, dictamen juridico, auditor oficial, aprobacion de Cabildo, garantia de ahorro, ni sistema autonomo que decida por el municipio.

## Que decir al mostrar benchmarks

> Este benchmark sirve como referencia comparativa. No es estudio local ni cifra municipal validada. Si se quiere usar una conclusion local, necesitamos evidencia municipal o estudio de campo.

## Que decir al mostrar inferencias

> Esta inferencia viene de fuente publica o calculo trazable. Debe conservar fuente, fecha, metodo y confianza. Hasta que un humano la valide, se trata como preliminar.

## Que decir al mostrar brechas criticas

> La brecha critica aparece para evitar sobreafirmar. Si falta un estudio local o evidencia minima, ALQUIMIA pausa el claim en vez de maquillarlo.

## Que decir al mostrar capa de aprendizaje supervisado

> capa de aprendizaje supervisado observa correcciones, outcomes y deltas. Solo puede sugerir con opt-in, N suficiente, bias check, founder gate y trazabilidad. No firma, no aprueba y no decide.

## Que decir al mostrar flujo de borradores asistidos

> flujo de borradores asistidos prepara lecturas y borradores a partir de evidencia. La aprobacion sigue siendo humana: fundador, juridico, Cabildo o responsable designado.

## Que decir si falta estudio local

> No vamos a llenar ese vacio con una narrativa. Si el claim requiere estudio local, queda como brecha critica hasta contar con alcance, fecha, metodo, fuente y responsable.

## Que no decir nunca

- "ALQUIMIA certifica resultados."
- "Esto ya esta validado oficialmente" si falta autoridad competente.
- "El benchmark prueba la realidad local."
- "capa de aprendizaje supervisado decidio."
- "flujo de borradores asistidos aprobo."
- "El ahorro/captura/reduccion esta garantizado."
- "La ZM sirve igual que el municipio."
- "Podemos cerrar el gate sin evidencia."

## Cierre recomendado por prospecto

| Prospecto | Cierre seguro |
|---|---|
| Municipio con pocos datos | "El siguiente paso correcto es diagnostico preliminar con brechas visibles." |
| Municipio con datos operativos | "El siguiente paso es diagnostico con datos provistos y revision humana." |
| Direccion tecnica | "El siguiente paso es mesa tecnica para validar fuentes, metodos y estudios faltantes." |
| Cabildo/regidor | "El siguiente paso es expediente preliminar, no dictamen, con evidencias y bloqueos claros." |
| Concesionario | "El siguiente paso es contrastar estimaciones contra evidencia operativa." |
| Empresa/institucion | "El siguiente paso es delimitar alcance, datos disponibles y claims permitidos." |
