# Safe Language Bank

**Uso:** frases permitidas, prohibidas y reformulaciones sobrias para conversaciones institucionales.

## Frases permitidas

- "ALQUIMIA ordena evidencia y escenarios; no sustituye decision publica."
- "Esta cifra es preliminar hasta que exista validacion humana."
- "Este benchmark orienta, pero no sustituye estudio local."
- "Esta inferencia requiere fuente, fecha, metodo y confianza."
- "Si falta estudio local, lo correcto es declarar brecha critica."
- "capa de aprendizaje supervisado observa y sugiere bajo controles; no decide."
- "flujo de borradores asistidos asiste en razonamiento y borradores; no aprueba."
- "Municipio y ZM deben mantenerse separados."

## Frases prohibidas

- "ALQUIMIA certifica resultados."
- "Esto esta oficialmente validado" sin autoridad competente.
- "El benchmark prueba el comportamiento municipal."
- "La estimacion es la verdad operativa."
- "capa de aprendizaje supervisado decidio."
- "flujo de borradores asistidos aprobo."
- "Podemos garantizar ahorro/captura/reduccion."
- "La ZM sirve igual que el municipio."

## Reformulaciones sobrias

| Si alguien quiere decir | Decir |
|---|---|
| "Garantizamos ahorro." | "Podemos modelar escenarios de ahorro con supuestos trazables." |
| "capa de aprendizaje supervisado predice." | "capa de aprendizaje supervisado observa patrones bajo controles y prepara sugerencias revisables." |
| "Ya esta validado." | "Esta documentado con fuente; falta o existe validacion humana segun el caso." |
| "El estudio no hace falta." | "Sin estudio local, el claim queda preliminar o bloqueado." |
| "La plataforma aprueba." | "La plataforma registra; la aprobacion es humana." |

## Como hablar de estimaciones

> Es una estimacion con supuestos visibles. Sirve para orientar, no para declarar una cifra oficial.

## Como hablar de benchmarks

> Es una referencia comparativa. No prueba la realidad local.

## Como hablar de capa de aprendizaje supervisado

> capa de aprendizaje supervisado no decide. Solo puede sugerir cuando hay opt-in, N suficiente, bias check, founder gate y trazabilidad.

## Como hablar de flujo de borradores asistidos

> flujo de borradores asistidos organiza evidencia y redacta borradores; no firma ni aprueba.

## Como hablar de impacto

> El impacto puede modelarse como escenario. Para presentarlo como resultado se requiere medicion y validacion.

## Como hablar de ahorro

> El ahorro debe expresarse como escenario o estimacion hasta tener baseline, fuente y validacion.

## Como hablar de captura

> La captura estimada depende de supuestos operativos; la captura real requiere medicion.

## Como hablar de incertidumbre

> La incertidumbre no se oculta. Se muestra como confianza baja, pendiente o brecha.

## Como hablar de brecha critica

> Brecha critica significa que falta evidencia necesaria para sostener el claim. No es un detalle menor.
