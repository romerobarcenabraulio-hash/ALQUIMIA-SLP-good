# Post Meeting Follow Up Playbook

**Uso:** seguimiento posterior a reunion institucional sin perder trazabilidad ni convertir interes en promesa.

## Primeras 24 horas

1. Registrar minuta con asistentes, fecha, institucion y tipo de reunion.
2. Listar claims mencionados y quien los menciono.
3. Separar evidencia mostrada de evidencia faltante.
4. Registrar datos que el municipio/institucion prometio enviar.
5. Registrar datos que NO deben asumirse.
6. Clasificar siguiente paso con `NEXT_STEP_DECISION_TREE.md`.
7. Registrar promesas o frases riesgosas en `PROMISE_CONTROL_REGISTER.md`.
8. Pausar si aparece riesgo metodologico P0.

## Que registrar

- Claims discutidos.
- Evidencia mostrada.
- Evidencia solicitada.
- Dudas abiertas.
- Riesgos politicos, tecnicos, juridicos o de concesionario.
- Datos prometidos por municipio/institucion.
- No-acuerdos: cosas conversadas que no son compromiso.
- Responsable humano y fecha de seguimiento.

## Datos que NO deben asumirse

- Toneladas reales sin medicion local.
- Composicion fisica sin cuarteo o estudio equivalente.
- Costo real sin fuente financiera.
- Postura de Cabildo sin fuente o validacion.
- Cumplimiento legal sin juridico.
- Ahorro/captura/reduccion sin baseline y metodo.
- Opt-in agregado sin consentimiento explicito.

## Clasificacion del siguiente paso

| Siguiente paso | Uso |
|---|---|
| Enviar informacion | Interes inicial sin datos. |
| Diagnostico preliminar | Aceptan advertencias y preliminaridad. |
| Diagnostico con datos provistos | Hay datos disponibles para revisar. |
| Piloto controlado | Hay sponsor, alcance y evidencia inicial. |
| Estudio local requerido | El claim depende de evidencia de campo. |
| Revision juridica | Hay documento, contrato, Cabildo o reforma. |
| Mesa tecnica | Hay disputa de fuente/metodo/alcance. |
| Pausa/no-fit | Exigen claims no sustentados. |

## Minuta post-reunion

| Campo | Contenido |
|---|---|
| Asistentes | Nombres, roles e institucion. |
| Institucion | Municipio, empresa o entidad. |
| Fecha | Fecha de reunion. |
| Tipo de reunion | Exploratoria, demo, Cabildo, mesa tecnica, concesionario, auditoria. |
| Claims discutidos | Afirmaciones mencionadas o solicitadas. |
| Evidencia mostrada | Fuente, fecha, metodo, documento o demo. |
| Evidencia faltante | Brechas y estudios requeridos. |
| Riesgos | Politico, juridico, tecnico, datos, concesionario, privacidad. |
| Acuerdos reales | Solo compromisos explicitos. |
| No-acuerdos | Interes o comentarios no vinculantes. |
| Siguiente paso | Clasificacion del arbol. |
| Responsable humano | Persona que valida o envia datos. |
| Fecha de seguimiento | Fecha acordada o propuesta. |

## Cuándo pausar

Pausar si el interlocutor pide ocultar brecha critica, vender benchmark como estudio local, usar estimacion como oficial, atribuir aprobacion a capa de aprendizaje supervisado/flujo de borradores asistidos, mezclar municipio/ZM o avanzar sin responsable humano.
