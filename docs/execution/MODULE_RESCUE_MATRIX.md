## Reglas De Eliminación

- Eliminar UI cliente que dependa de sliders libres, SLP hardcodeado, `municipio-demo` con cifras, nombres internos de agentes o benchmark como dato local.
- Cuarentenar componentes founder/admin útiles que todavía dependan de `simulatorStore`.
- Rescatar sólo lógica visual o matemática que pueda recibir props desde `tenant_id`, `EvidenceKernel`, `CityConsultingContext` o datos trazables.
