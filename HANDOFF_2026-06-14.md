## 3. Stripe — cómo está conectado

### Variables de entorno requeridas (Render + Vercel)
```
SECRET_KEY_STRIPE          # sk_live_... — solo en Render (backend), NUNCA en Vercel
PUBLIC_STRIPE_KEY          # pk_live_... — en Vercel (frontend) y Render
STRIPE_WEBHOOK_SECRET      # whsec_... — solo en Render
```

### Flujo actual implementado
```
Frontend (Vercel)
  └─ GET /api/v1/payments/config  →  devuelve PUBLIC_STRIPE_KEY
  └─ POST /api/v1/payments/checkout (crea PaymentIntent vía Stripe)
  └─ POST /api/v1/stripe/webhook  →  router stripe_webhooks.py
                                      ├─ verifica firma STRIPE_WEBHOOK_SECRET
                                      ├─ actualiza Subscription en DB
                                      └─ crea Invoice / Transaction
```

### Archivos clave
| Archivo | Rol |
|---------|-----|
| `app/config.py` líneas 26-29 | Lee las 3 env vars |
| `app/config.py` líneas 44-46 | Inicializa `stripe.api_key` |
| `app/routers/payments.py` | CRUD subscriptions / invoices (solo admin) |
| `app/routers/stripe_webhooks.py` | Webhook handler + verificación de firma |
| `app/models/payment.py` | Modelos: PaymentMethod, Subscription, Invoice, Transaction |

### Para activar en Render
1. Settings → Environment → Add:
   - `SECRET_KEY_STRIPE=sk_live_...`
   - `PUBLIC_STRIPE_KEY=pk_live_...`
   - `STRIPE_WEBHOOK_SECRET=whsec_...`
2. En el dashboard de Stripe → Webhooks → endpoint: `https://tu-app.onrender.com/api/v1/stripe/webhook`
3. Eventos a escuchar: `payment_intent.succeeded`, `invoice.paid`, `customer.subscription.updated`, `customer.subscription.deleted`

### Para activar en Vercel
Solo necesita `PUBLIC_STRIPE_KEY` (clave pública, no secreta):
- Settings → Environment Variables → `NEXT_PUBLIC_STRIPE_KEY=pk_live_...`  
  *(o `PUBLIC_STRIPE_KEY` dependiendo de cómo el frontend la consume)*
